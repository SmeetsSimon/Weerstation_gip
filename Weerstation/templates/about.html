Gip samenvatting

Ik heb een weerstation gemaakt die de temperatuur, luchtvochtigheid en de luchtdruk kan meten met een sensor deze sensor laat daarna alle gegevens zien op een lcd scherm. Voor dit te programmeren heb ik een Raspberry pi gebruikt. En als laatste moeten de gegevens live te zien zijn op mijn website.

Sensor

Ik heb de BME280 sensor van Adafruit gebruikt. Met deze sensor kan je zeer nauwkeurig de temperatuur, luchtdruk en luchtvochtigheid.
Er zijn 2 verschillende soorten aansluiten, ik maak gebruik van i2c dat heb ik gedaan omdat de LCD die ik gebruik hier ook mee werkt. Bij i2c maak je gebruik van 4 aansluitpinnen: Vin, GND, SCK en SDI. De Vin en GND zijn vanzelfsprekend maar de SCK en de SDI waren voor mij nieuw. SCK staat voor 'SPI Clock Pin' dit is de input naar de chip en word aangesloten op de SCL pin. SDI staat voor 'Serial Data In' via deze pin verzend de processor data naar de BME280 en word aangesloten op de SDA pin. Zie hier de code die ik heb geschreven voor de Sensor.

Hier kun je zien hoe de BME280 aan de Raspberry Pi is aangesloten

LCD

Ik gebruik een 20X4 LCD van JOY-IT. De LCD heeft een I2D communicatie-adapter er achterop gesoldeerd. Dat is handig want het heeft de zelfde werking als bij de sensor. 4 pinnen: GND, VCC, SDA en SCL. De SCL komt overeen met de SDA die we terug hebben gezien op de sensor. De SCL is hetzelfde als de SCK. Zie hier de code die ik heb geschreven voor de LCD.

Raspberry Pi

Voor de programmeren heb ik gekozen voor een Raspberry Pi. Een Raspberry Pi is een microcontroller je kan het vergelijken met een kleine pc. je programmeerd hier gewoon met python op daarom dat ik ervoor heb gekozen in plaats van een Arduino die programmeerd met C++.

Opbouwen gip

Voor het visueel aantrekkelijk te maken heb ik een doosje laten maken waar ik mijn Raspberry Pi, Sensor en LCD heb ik zitten. het doosje is gemaakt van hout dat ik heb laten lazeren op school.

Data logging

De opdracht was om de gegevens te laten tonen op een LCD maar ook op een website. Dit kan je zien in mijn Dashboard pagina. Voor dit te doen had ik speciaal gekozen voor Django maar dat ging me veel te veel tijd kosten om te doen en dan zou ik mijn website nooit afkrijgen. Daarom heb ik gekozen voor ThingSpeak.
ThingSpeak is een service waarmee je live gegevensstromen in de cloud kunt samenvoegen, visualiseren en analyseren. Voor dit toe te voegen aan mijn code moest ik een paar regels toevoegen Zie Code. Als ik mijn programma ging laten draaien stuurde die de gegevens van de sensor door naar de cloud van ThingSpeak. Dan kun je live de gegevens zien binnenkomen op ThingSpeak. Daarna had ik de grafieken die ik had gemaakt op mijn website gezet waardoor je ze ook live kunt zien op mijn website.

Mijn website is een belangrijk deel van mijn gip. Ik moest in plaats van een bundel een website maken en daar bovenop moet ik ook nog een grafische weergave tonen van de sensor zijn gegevens. Ik had hier kunnen kiezen tussen html en Django en ik heb voor Django gekozen. Die keuze had ik gemaakt met in gedachten een database te maken met via Django want dat ging niet met gewoon Html.
Later ben ik er dan achter gekomen wat ThingSpeak was en had ik ook nog de keuze om opnieuw te beginnen met html. Dit was toen ik al ver was met mijn website op Django maar ik botste telkens tegen nieuwe problemen, omdat Django uitgebreider en moeilijker is dan Html. Ik ben toch verder gegaan met Django omdat het me na een half jaar ermee te werken wel beviel. Nu uiteindelijk heb ik wel gebruik gemaakt van Django met mijn Contact pagina.
